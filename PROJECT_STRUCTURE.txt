===============================================================
DATABASE SCHEMA (SQLite - hospital.db)
===============================================================

devices TABLE
├── id (INTEGER PRIMARY KEY)
├── device_id (TEXT UNIQUE) → "ESP32_BA_F2_R201_BED1"
├── building (TEXT) → "A", "B", "C"
├── floor (INTEGER) → 1, 2, 3
├── room (TEXT) → "201", "202"
├── bed (INTEGER) → 1, 2, 3
├── last_seen (TEXT) → ISO8601
├── battery (INTEGER) → 0-100%
├── created_at (TEXT)
└── updated_at (TEXT)

vitals TABLE
├── id (INTEGER PRIMARY KEY)
├── device_id (TEXT FK → devices)
├── heart_rate (INTEGER) → bpm
├── spo2 (REAL) → percentage
├── temperature (REAL) → celsius
├── bp_sys (INTEGER) → mmHg
├── bp_dia (INTEGER) → mmHg
└── recorded_at (TEXT) → ISO8601

alerts TABLE
├── id (INTEGER PRIMARY KEY)
├── device_id (TEXT FK → devices)
├── type (TEXT) → "CRITICAL", "WARNING", "INFO"
├── message (TEXT)
├── acknowledged (INTEGER) → 0/1
└── created_at (TEXT) → ISO8601

patients TABLE
├── id (INTEGER PRIMARY KEY)
├── device_id (TEXT UNIQUE FK → devices)
├── name (TEXT)
├── age (INTEGER)
├── diagnosis (TEXT)
├── admitted_at (TEXT) → ISO8601
├── created_at (TEXT)
└── updated_at (TEXT)


===============================================================
API ENDPOINTS (REST)
===============================================================

POST   /api/data
       Input: {device_id, building, floor, room, bed, vitals, alerts, battery}
       Action: Create/update device, insert vitals, broadcast WebSocket

GET    /api/buildings
       Output: Hierarchy: {building: {floor: {room: [devices with vitals]}}}
       Use: Dashboard building view

GET    /api/devices
       Output: [device] with latestVital, patient, unreadAlerts
       Use: Device listing, status monitoring

GET    /api/bed/:device_id/vitals?limit=50
       Output: Vital history in chronological order
       Use: BedDetail page charts

GET    /api/alerts?type=CRITICAL&unread=true
       Output: [alerts] with device info
       Use: Alerts Center filtering

GET    /api/alerts/stats
       Output: {total, unread, critical, warning, info}
       Use: Alert statistics display

POST   /api/alerts/:id/acknowledge
       Action: Mark alert as read, broadcast WebSocket
       Output: {success, unreadCount}
       Use: Alert acknowledgment button

GET    /api/patients
       Output: [patient] ordered by name
       Use: Patient listing in Settings

POST   /api/patients
       Input: {device_id, name, age, diagnosis}
       Action: Create/update patient
       Output: {success, patient}
       Use: Patient form submission

DELETE /api/patients/:device_id
       Action: Delete patient record
       Output: {success, message}
       Use: Delete button in patient table

GET    /api/health
       Output: {status: "ok", timestamp}
       Use: Health check, simulator readiness check


===============================================================
WEBSOCKET EVENTS
===============================================================

Server → Clients:

new_vitals
├── device_id
├── building, floor, room, bed
├── vitals {heart_rate, spo2, temperature, bp_sys, bp_dia, battery}
└── timestamp
Use: Dashboard & charts real-time updates

new_alert
├── device_id
├── type (CRITICAL, WARNING, INFO)
├── message
├── building, floor, room, bed
└── timestamp
Use: Toast notification, alert list update

alert_acknowledged
├── alert_id
└── unreadCount
Use: Update sidebar badge, alert list refresh
